# AI Food Waste Forecasting

## Problem
Restaurants often prepare more food than needed, which leads to unnecessary food waste, higher costs, and avoidable environmental impact.

## Solution
Build an AI/ML system that predicts daily demand for each dish so restaurants can prepare closer to the right amount.

## How It Works
- Uses historical sales, day of week, weather, and local events
- Predicts how much of each dish will likely be sold
- Recommends daily prep quantities to reduce overproduction

## Goal
Help restaurants cut food waste, improve planning, and operate more sustainably.

## Repo Structure

```text
.
├── data/
│   ├── raw/
│   │   ├── inventory/
│   │   ├── recipes/
│   │   ├── sales/
│   │   ├── waste/
│   │   └── weather/
│   └── processed/
├── scripts/
│   └── data/
└── README.md
```

- `data/raw/`: source datasets exactly as they are collected or downloaded.
- `data/processed/`: cleaned datasets generated by scripts for modeling or app use.
- `scripts/data/`: one-off or repeatable ingestion/normalization utilities.

## Current Files

- Weather data lives in `data/raw/weather/toronto_weather_2026.csv`.
- Recipe files live in `data/raw/recipes/`.
- Inventory data lives in `data/raw/inventory/`.
- Food waste data lives in `data/raw/waste/`.
- The weather fetch script is `scripts/data/fetch_weather.py`.
- The sales normalization script is `scripts/data/normalize_sales.py`.
